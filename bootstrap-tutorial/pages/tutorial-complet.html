Installation de Bootstrap

Il existe plusieurs façons d'intégrer Bootstrap dans votre projet :

1. Via CDN

La méthode la plus rapide pour commencer avec Bootstrap est d'utiliser le CDN :

<!-- CSS Bootstrap --> <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet"> <!-- JavaScript Bootstrap Bundle avec Popper --> <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>

2. Via npm

npm install bootstrap@5.3.2

3. Téléchargement direct

Vous pouvez également télécharger Bootstrap directement depuis le site officiel.

Espacement

Personnalisation de Bootstrap

Vous pouvez personnaliser Bootstrap de différentes façons :

1. Variables CSS

:root { --bs-primary: #0066cc; /* Remplace la couleur primaire */ --bs-border-radius: 0.5rem; /* Arrondis plus importants */ --bs-font-sans-serif: "Helvetica Neue", Helvetica, Arial, sans-serif; }

// Personnalisation des variables $primary: #0066cc; $border-radius: 0.5rem; $font-family-sans-serif: "Helvetica Neue", Helvetica, Arial, sans-serif; // Import de Bootstrap @import "node_modules/bootstrap/scss/bootstrap";

Bonnes Pratiques et Projets Complets

Apprenez à structurer vos projets et à appliquer les meilleures pratiques avec Bootstrap.

Une bonne organisation du code est essentielle pour maintenir des projets Bootstrap à long terme.

Structure de dossiers recommandée

my-bootstrap-project/ ├── assets/ │ ├── css/ │ │ ├── custom.css # Votre CSS personnalisé │ │ └── bootstrap.min.css # Version minifiée de Bootstrap (si pas via CDN) │ ├── js/ │ │ ├── custom.js # Votre JavaScript personnalisé │ │ └── bootstrap.bundle.min.js # Bootstrap JS (si pas via CDN) │ └── img/ # Images du site ├── components/ # Fragments HTML réutilisables │ ├── navbar.html │ ├── footer.html │ └── sidebar.html ├── pages/ # Pages individuelles │ ├── about.html │ ├── contact.html │ └── products.html └── index.html # Page d'accueil

Conventions de nommage

  • Utilisez des noms de classes descriptifs qui expliquent leur fonction (ex: product-card plutôt que card1)
  • Préfixez vos classes personnalisées pour éviter les conflits avec Bootstrap (ex: my-custom-btn)
  • Restez cohérent dans votre style de nommage (kebab-case est recommandé pour HTML/CSS)

Séparation des préoccupations

Séparez clairement votre structure (HTML), votre présentation (CSS) et votre comportement (JavaScript).

<!-- Bon exemple --> <button id="saveBtn" class="btn btn-primary">Enregistrer</button> <style> /* CSS séparé pour la présentation */ .btn-primary { text-transform: uppercase; } </style> <script> // JavaScript séparé pour le comportement document.getElementById('saveBtn').addEventListener('click', function() { saveData(); }); </script> <!-- Mauvais exemple --> <button onclick="saveData()" style="text-transform: uppercase;" class="btn btn-primary">Enregistrer</button>

Optimisez vos projets Bootstrap pour de meilleures performances.

Minimiser les imports

Bootstrap est modulaire. N'importez que ce dont vous avez besoin pour réduire la taille du fichier.

// Si vous utilisez Sass, n'importez que les composants nécessaires // _custom.scss @import "~bootstrap/scss/functions"; @import "~bootstrap/scss/variables"; @import "~bootstrap/scss/mixins"; // Composants requis @import "~bootstrap/scss/reboot"; @import "~bootstrap/scss/grid"; @import "~bootstrap/scss/buttons"; @import "~bootstrap/scss/card"; // Uniquement les composants que vous utilisez

Optimiser les images

  • Utilisez des formats d'image appropriés (WebP, SVG pour les icônes)
  • Compressez les images pour le web
  • Utilisez la classe img-fluid pour les images responsives
  • Utilisez le lazy loading pour les images
<img src="image.jpg" class="img-fluid" alt="..." loading="lazy">

Éviter les manipulations DOM inutiles

Minimisez les manipulations DOM avec JavaScript. Utilisez la délégation d'événements quand c'est possible.

// Mauvais - Attachement d'événements individuels à chaque bouton document.querySelectorAll('.btn').forEach(button => { button.addEventListener('click', handleClick); }); // Bon - Utilisation de la délégation d'événements document.querySelector('.container').addEventListener('click', function(e) { if (e.target.matches('.btn')) { handleClick(e); } });

Voici la structure recommandée pour un projet complet avec Bootstrap.

Structure de fichiers

projet-bootstrap/ ├── assets/ │ ├── css/ │ │ ├── custom.css │ │ └── vendor/ # CSS de bibliothèques tierces │ ├── js/ │ │ ├── main.js │ │ └── vendor/ # JS de bibliothèques tierces │ ├── img/ │ └── fonts/ ├── components/ # Fragments HTML réutilisables ├── layouts/ # Mises en page différentes ├── pages/ # Pages individuelles ├── scss/ # Sources Sass │ ├── custom/ # Variables et surcharges personnalisées │ ├── _variables.scss # Variables Bootstrap personnalisées │ └── main.scss # Point d'entrée principal ├── .gitignore ├── package.json # Dépendances npm ├── webpack.config.js # Configuration de build (si utilisé) └── index.html
