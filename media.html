Media Queries CSS

Introduction

Les media queries permettent d'appliquer des styles CSS conditionnellement en fonction des caractéristiques de l'appareil, comme la largeur d'écran, l'orientation ou le type d'appareil. Cette démonstration interactive vous aidera à comprendre leur fonctionnement et leurs applications pour créer des sites web responsifs.

Détection de la largeur d'écran

Les media queries les plus courantes sont basées sur la largeur d'écran. Elles permettent d'adapter votre design aux différentes tailles d'écran.

Vous êtes sur un appareil mobile (max-width: 767px)
Vous êtes sur une tablette (min-width: 768px)
Vous êtes sur un ordinateur de bureau (min-width: 1024px)
Vous êtes sur un grand écran (min-width: 1440px)

Redimensionnez votre navigateur pour voir comment le message change en fonction de la largeur de l'écran. Ceci est réalisé grâce à des media queries basées sur la largeur.

/* Mobile */ @media only screen and (max-width: 767px) { #mobile-info { display: block; } } /* Tablette */ @media only screen and (min-width: 768px) and (max-width: 1023px) { #tablet-info { display: block; } } /* Desktop */ @media only screen and (min-width: 1024px) and (max-width: 1439px) { #desktop-info { display: block; } } /* Large Desktop */ @media only screen and (min-width: 1440px) { #large-info { display: block; } }

Navigation responsive

Un modèle courant est de transformer une barre de navigation horizontale en menu hamburger sur mobile.

  • Accueil
  • Produits
  • Services
  • À propos
  • Contact

Redimensionnez votre navigateur pour voir la navigation changer. Sur petit écran, elle devient un menu hamburger. Cliquez sur "Menu" pour l'ouvrir/fermer.

/* Navigation par défaut */ .nav-toggle { display: none; } .nav-list { display: flex; list-style: none; } /* Navigation mobile */ @media only screen and (max-width: 767px) { .nav-toggle { display: block; } .nav-list { display: none; flex-direction: column; } .nav-list.active { display: flex; } }

Images responsives

Les images peuvent être rendues responsives pour s'adapter à différentes tailles d'écran.

Cette image s'adapte à la largeur de son conteneur, tout en maintenant son ratio hauteur/largeur. Redimensionnez votre navigateur pour voir l'effet.

.responsive-image { max-width: 100%; height: auto; display: block; }

Texte responsive

La taille du texte peut également être ajustée en fonction de la taille de l'écran.

Titre responsive

Ce paragraphe change de taille en fonction de la largeur de l'écran. Sur mobile, il est plus petit. Sur desktop, il est plus grand. Redimensionnez votre navigateur pour voir la différence.

La taille du texte s'adapte à différentes tailles d'écran pour une meilleure lisibilité.

/* Taille par défaut (mobile) */ .responsive-text h2 { font-size: 24px; } .responsive-text p { font-size: 16px; line-height: 1.6; } /* Tablette et plus */ @media only screen and (min-width: 768px) { .responsive-text h2 { font-size: 32px; } .responsive-text p { font-size: 18px; line-height: 1.8; } } /* Desktop et plus */ @media only screen and (min-width: 1024px) { .responsive-text h2 { font-size: 36px; } .responsive-text p { font-size: 20px; line-height: 2; } }

Media queries d'orientation

Les media queries peuvent également cibler l'orientation de l'appareil (portrait ou paysage).

Orientation actuelle:

MODE PORTRAIT DÉTECTÉ

Tournez votre appareil ou redimensionnez votre navigateur pour voir le mode paysage.

MODE PAYSAGE DÉTECTÉ

Tournez votre appareil ou redimensionnez votre navigateur pour voir le mode portrait.

Cette démo détecte l'orientation de votre écran et affiche un message différent en fonction de celle-ci.

@media (orientation: portrait) { .portrait-content { display: block; } } @media (orientation: landscape) { .landscape-content { display: block; } }

Media features avancées

Au-delà de la largeur et de l'orientation, les media queries peuvent détecter d'autres caractéristiques comme le type d'appareil, les préférences de l'utilisateur, etc.

Vous utilisez un écran tactile (hover: none)

Vous utilisez un dispositif de pointage précis comme une souris (hover: hover)

Vous préférez le thème sombre (prefers-color-scheme: dark)

Vous préférez le thème clair (prefers-color-scheme: light)

Ces exemples montrent comment détecter des caractéristiques avancées comme le type d'appareil (tactile ou non) et les préférences de thème de l'utilisateur.

/* Écran tactile */ @media (hover: none) and (pointer: coarse) { #touch-screen { display: block; } } /* Écran non tactile avec souris/trackpad */ @media (hover: hover) and (pointer: fine) { #mouse-screen { display: block; } } /* Préférence de thème */ @media (prefers-color-scheme: dark) { #dark-theme { display: block; } } @media (prefers-color-scheme: light) { #light-theme { display: block; } }

Récapitulatif

Les media queries sont essentielles pour créer des sites web qui s'adaptent à tous les appareils et préférences utilisateur.

Syntaxe de base :

@media mediaType and (mediaFeature: value) {
    /* Styles à appliquer si la condition est vraie */
}

Types de media courants :

  • all - Tous les appareils
  • screen - Écrans
  • print - Impression
  • speech - Synthèse vocale

Media features courantes :

  • width, min-width, max-width - Largeur de la fenêtre
  • height, min-height, max-height - Hauteur de la fenêtre
  • orientation - Portrait ou paysage
  • aspect-ratio - Ratio largeur/hauteur
  • resolution - Densité de pixels
  • hover - Capacité de survol
  • pointer - Type de dispositif de pointage
  • prefers-color-scheme - Préférence de thème (clair/sombre)
  • prefers-reduced-motion - Préférence de réduction des animations

Opérateurs logiques :

  • and - Toutes les conditions doivent être vraies
  • not - Inverse la condition
  • , (virgule) - Équivalent à OR, une des conditions doit être vraie
